Frequently asked questions

How SENTIO works, what it proves and what it does not claim.

SENTIO is designed as a deployable environmental intelligence network: simple sensing capsules in the field, interpretation in software and confidence built from distributed evidence.

FAQ

Clear answers for technical, operational and startup conversations.

These answers describe the current concept and software pilot. Hardware claims are intentionally framed as prototype direction, not certified field performance.

What is SENTIO?

SENTIO is an attrition-tolerant environmental intelligence network for places where fixed monitoring infrastructure is unavailable, slow to install or unsafe to access. It combines low-cost sensing capsules, deployable gateways and software-defined intelligence modules.

What does "attrition tolerant by design" mean?

The network does not depend on every capsule surviving, charging, transmitting or being recovered. Capsule loss is treated as an expected operating condition. The system is designed around redundancy, spatial coverage and network-level confidence rather than perfect individual node survival.

What does the capsule actually do?

The capsule is a data collection node. It collects compact environmental, acoustic, vibration, battery, solar and radio-health telemetry. It is not intended to make final mission decisions by itself. Event interpretation happens in the software layer.

Why is the intelligence in software instead of in every capsule?

Keeping the capsule simple lowers unit cost, reduces loss risk and limits exposure if a capsule is damaged, lost or reverse-engineered. The software layer can combine readings across many capsules, update interpretation modules and score confidence using wider network context.

What is SENTIO v0?

SENTIO v0 is the software pilot. It simulates a live capsule network, streams telemetry through a FastAPI backend, applies River and scikit-learn anomaly models, calculates network confidence and displays the results in a live dashboard with maps, charts, exports and explainable alerts.

Does SENTIO v0 use real sensors?

SENTIO v0 currently demonstrates the end-to-end software workflow with simulated pods and a live packet tester. The packet tester is designed so future physical nodes can send compatible JSON telemetry into the same processing path.

How is incident confidence calculated?

SENTIO treats a single abnormal reading as weak evidence. Confidence rises when local anomaly evidence, model evidence, nearby capsule agreement and node health support the same interpretation. The simplified logic is: blended anomaly evidence multiplied by neighbour validation and discounted by node health.

What AI modules could run on the same infrastructure?

The same capsule infrastructure can support different software modules, including wildfire monitoring, pollution detection, SAR baseline awareness, biodiversity monitoring, infrastructure stress monitoring and illegal logging indicators. The commercial idea is one physical network with selectable intelligence modules.

Is search-and-rescue a core claim?

SENTIO frames SAR as a support layer, not as a precise person-identification system. The goal is to help prioritise search areas using human-activity or environmental disturbance indicators while avoiding overclaiming exact individual location or identity.

Why not just install expensive permanent stations?

Permanent stations can be accurate but sparse, expensive and slow to deploy. SENTIO is aimed at rapid situational coverage across remote, hazardous or temporary zones where perfect installation and perfect recovery cannot be assumed.

What happens if capsules are lost or offline?

Missing nodes reduce coverage but do not automatically break the network. SENTIO v0 separates missing or offline status from incident confidence, so a deployment can still produce useful evidence from the remaining active capsules.

What is the next validation step after SENTIO v0?

The next step is a live node: one physical sensor package sending SENTIO-compatible telemetry into the software pipeline. After that, the roadmap moves toward a small gateway deployment, outdoor power and enclosure validation, and labelled field tests.
